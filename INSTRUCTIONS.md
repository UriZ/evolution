# Implementation Instructions

## Code Philosophy

1. **Less is more**: Keep code short and concise. No fluff, no unnecessary abstractions.

2. **Persistence**: If you can't solve a problem, try again with the same approach before switching to a completely new technology/method. Give it 2-3 attempts before giving up on an approach.

3. **Small incremental changes**: Work in small chunks. Test after each change.

4. **Read before modify**: Always read files before editing them.

5. **Test as you go**: Verify each component works before moving to the next.
